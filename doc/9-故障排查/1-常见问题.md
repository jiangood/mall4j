# 常见问题

## 管理后台接口 401

优先检查：

- `front-end/mall4v/.env.development` 是否指向 `http://127.0.0.1:8085`。
- `yami-shop-admin` 是否已启动。
- 浏览器里是否有过期的 `Authorization`，必要时退出重新登录。

如果登录后提示无权限，也先确认管理后台没有误连用户端接口。管理后台默认端口是 `8085`，用户端接口默认端口是 `8086`。

## 验证码或登录接口异常

优先检查管理后台是否连到了用户端 `8086`。管理后台必须连接 `8085`。

## 小程序或 uni-app 接口异常

小程序和 uni-app 应连接用户端接口：

```text
http://127.0.0.1:8086
```

配置位置：

```text
front-end/mall4m/utils/config.js
front-end/mall4uni/.env.development
```

## 跨域问题

浏览器控制台出现 `Access-Control-Allow-Origin`，或 Network 中 `OPTIONS` 请求失败，通常是跨域。

处理方向：

- 本地开发先确认前端接口地址和后端端口一致。
- 生产环境建议用 Nginx 反向代理，把同一域名下的不同路径转发到后端服务。
- 如果使用 `/apis` 这类相对路径，必须同时配置 Nginx rewrite 和 `proxy_pass`。

## 图片不显示

上传成功但页面图片不显示时，优先检查管理后台环境变量：

```text
VITE_APP_RESOURCES_URL
```

它必须指向实际可访问的静态资源域名或本地资源映射地址。旧文档里的 `VUE_APP_RESOURCES_URL` 是历史命名，当前 `mall4v` 应使用 `VITE_APP_RESOURCES_URL`。

## 页面能进但按钮不显示

按钮由权限控制。检查：

- `tz_sys_menu` 是否有对应按钮权限。
- 角色是否分配了该按钮权限。
- 前端 `isAuth()` 使用的权限标识是否和后端一致。

## 菜单不显示

检查：

- 是否新增了 `type=1` 的菜单。
- 菜单 `url` 是否正确。
- 当前角色是否分配该菜单。
- 前端页面路径是否和菜单 `url` 对应。

例如：

```text
菜单 url: shop/helpArticle
页面路径: front-end/mall4v/src/views/modules/shop/helpArticle/index.vue
```
