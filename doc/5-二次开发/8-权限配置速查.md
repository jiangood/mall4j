# 权限配置速查

后台权限要同时经过后端、数据库、前端、角色分配四关。少一关，功能就可能“写了但看不见”。

## 四个位置

| 位置 | 作用 | 示例 |
| --- | --- | --- |
| 后端 `@PreAuthorize` | 控制接口能不能访问。 | `shop:helpArticle:save` |
| `tz_sys_menu.perms` | 存储按钮权限标识。 | `shop:helpArticle:save` |
| 前端 `isAuth()` | 控制按钮显示。 | `isAuth('shop:helpArticle:save')` |
| 角色授权 | 决定用户是否拥有权限。 | 给角色勾选菜单和按钮 |

## 菜单和按钮

菜单：

```text
type = 1
url = shop/helpArticle
```

按钮：

```text
type = 2
perms = shop:helpArticle:save
```

查看权限可以合并：

```text
shop:helpArticle:info,shop:helpArticle:page
```

这个写法在当前 `shop/notice` 菜单数据中已有类似用法。

## 完成后应该看到

- 登录后 `/sys/menu/nav` 返回新菜单。
- `sessionStorage` 中的 `Authorities` 包含对应按钮权限。
- 有权限用户能看到按钮。
- 无权限用户看不到按钮。
- 直接访问无权限接口时会返回未授权。

## 检查题-含答案

问题：菜单不显示，应该先查按钮权限还是菜单权限？

答案：先查菜单权限。菜单由 `type=1` 的菜单记录和角色授权决定；按钮权限只影响新增、修改、删除等按钮。

问题：按钮不显示，但接口能访问，最可能缺什么？

答案：前端 `isAuth()` 使用的权限标识和当前用户 `Authorities` 不一致，或角色没有分配按钮权限。

问题：接口 401，但按钮能看到，说明什么？

答案：前端按钮权限和后端接口权限可能不一致，或者当前登录状态/Token 已失效。先比对 `isAuth()`、`@PreAuthorize`、`tz_sys_menu.perms` 三处权限标识。
