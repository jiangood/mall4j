# 二开文件位置速查

新增后台管理功能时，先按这张表找位置。

| 你要做的事 | 放在哪里 | 参考 |
| --- | --- | --- |
| 新增数据库表 | `db/yami_shop.sql` 或单独迁移脚本 | `tz_notice`、`tz_sys_menu` |
| 新增实体 | `yami-shop-bean/src/main/java/com/yami/shop/bean/model/` | `Notice` |
| 新增 Mapper 接口 | `yami-shop-service/src/main/java/com/yami/shop/dao/` | `NoticeMapper` |
| 新增 Mapper XML | `yami-shop-service/src/main/resources/mapper/` | `NoticeMapper.xml` |
| 新增 Service 接口 | `yami-shop-service/src/main/java/com/yami/shop/service/` | `NoticeService` |
| 新增 Service 实现 | `yami-shop-service/src/main/java/com/yami/shop/service/impl/` | `NoticeServiceImpl` |
| 新增管理端接口 | `yami-shop-admin/src/main/java/com/yami/shop/admin/controller/` | `NoticeController` |
| 新增表格配置 | `front-end/mall4v/src/crud/` | `src/crud/shop/notice.js` |
| 新增页面 | `front-end/mall4v/src/views/modules/` | `src/views/modules/shop/notice/index.vue` |
| 新增菜单权限 | `tz_sys_menu` | `shop/notice`、`shop:notice:*` |

## 路径映射规则

菜单 URL 决定前端页面目录：

```text
菜单 url: shop/helpArticle
页面路径: front-end/mall4v/src/views/modules/shop/helpArticle/index.vue
```

接口路径建议和菜单 URL 保持同一业务命名：

```text
后端路径: /shop/helpArticle
前端请求: /shop/helpArticle/page
```

## 检查题-含答案

问题：菜单 `url` 写成 `shop/helpArticle` 时，前端默认加载哪个文件？

答案：`front-end/mall4v/src/views/modules/shop/helpArticle/index.vue`。

问题：后端管理接口应该放在 `yami-shop-admin` 还是 `yami-shop-api`？

答案：后台管理功能放在 `yami-shop-admin`。`yami-shop-api` 面向用户端、小程序、H5、uni-app。

问题：为什么二开后台功能不建议直接从订单或支付开始？

答案：订单、支付、库存属于交易主链路，风险高、状态多。新二开者应先从低风险后台 CRUD 学会项目结构和权限规则。
