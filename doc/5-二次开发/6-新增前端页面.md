# 新增前端页面

这一篇只关注管理后台页面。目标是让菜单能加载页面，页面能调用后端接口并展示数据。

管理后台页面位于：

```text
front-end/mall4v/src/views/modules/
```

菜单 URL 和页面路径需要对应。

```text
菜单 url: shop/helpArticle
页面路径: src/views/modules/shop/helpArticle/index.vue
```

## 推荐文件

```text
front-end/mall4v/src/crud/shop/helpArticle.js
front-end/mall4v/src/views/modules/shop/helpArticle/index.vue
front-end/mall4v/src/views/modules/shop/helpArticle/add-or-update.vue
```

## 列表页职责

`index.vue` 通常负责：

- 查询分页列表。
- 处理搜索。
- 打开新增/修改弹窗。
- 删除数据。
- 根据 `isAuth()` 控制按钮显示。

## 表单页职责

`add-or-update.vue` 通常负责：

- 初始化表单。
- 编辑时根据 ID 查询详情。
- 校验表单。
- 提交新增或修改请求。

## CRUD 配置

`src/crud/**` 里的配置用于描述表格字段、搜索项、下拉字典等。简单管理页面优先复用这种方式，不要一开始就手写复杂表格。

## 完成后应该看到

- 点击菜单后能进入页面。
- 列表页能请求 `/shop/helpArticle/page`。
- 表格能展示 `records`。
- 分页总数能显示 `total`。
- 新增和修改弹窗能打开。
- 没有权限的按钮不会显示。

## 检查题-含答案

问题：菜单 URL 和页面目录为什么必须一致？

答案：管理后台会根据菜单 `url` 动态加载 `src/views/modules/{url}/index.vue`。路径不一致会导致页面加载失败。

问题：为什么简单后台页面优先用 `src/crud/**` 配置？

答案：项目已有列表页使用这种模式描述表格字段、搜索项和字典，复用它可以减少重复代码并保持界面风格一致。

问题：页面能打开但列表没数据，应该先查什么？

答案：先查浏览器 Network 中分页接口是否成功，再查返回的 `records`、`total`，最后查查询条件和数据库数据。
