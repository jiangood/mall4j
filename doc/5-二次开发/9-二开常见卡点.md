# 二开常见卡点

这篇按现象反推排查位置。

## 菜单不显示

优先检查：

- `tz_sys_menu` 是否新增 `type=1` 菜单。
- 菜单 `url` 是否和前端页面目录一致。
- 当前角色是否分配了这个菜单。
- 重新登录后 `/sys/menu/nav` 是否返回新菜单。

正确示例：

```text
url = shop/helpArticle
页面 = front-end/mall4v/src/views/modules/shop/helpArticle/index.vue
```

## 页面空白或路由找不到

优先检查：

- 页面目录是否放在 `front-end/mall4v/src/views/modules/` 下。
- 是否存在 `index.vue`。
- 菜单 `url` 是否多写或少写 `/`。
- 浏览器控制台是否有动态 import 失败。

## 按钮不显示

优先检查：

- `tz_sys_menu` 是否新增 `type=2` 按钮权限。
- 当前角色是否分配了按钮权限。
- 前端 `isAuth()` 权限标识是否和 `tz_sys_menu.perms` 一致。
- 登录后 `sessionStorage` 的 `Authorities` 是否包含该权限。

## 接口 401

优先检查：

- 管理后台是否连接 `yami-shop-admin`，默认 `8085`。
- `Authorization` 是否过期。
- 后端 `@PreAuthorize` 权限标识是否和角色权限一致。
- 前端是不是误连用户端接口 `8086`。

## 分页没数据

优先检查：

- 前端是否传了 `current` 和 `size`。
- 后端是否返回 `records` 和 `total`。
- 查询条件是否带了当前店铺 `shopId`。
- 数据库是否真的有当前店铺的数据。

## 图片不显示

优先检查：

- 上传接口是否返回了文件路径。
- 前端 `VITE_APP_RESOURCES_URL` 是否指向可访问资源域名。
- 本地上传时后端资源映射是否能访问 `/mall4j/img/**`。

## 检查题-含答案

问题：菜单出现了，但点击后页面空白，最先查哪里？

答案：先查菜单 `url` 和前端页面目录是否一致，并确认该目录下有 `index.vue`。

问题：新增按钮没有出现，但删除按钮能出现，说明什么？

答案：当前用户有删除权限，但没有新增权限；检查 `shop:helpArticle:save` 是否写入 `tz_sys_menu` 并分配给角色。

问题：分页接口返回成功但列表空，前端一定错了吗？

答案：不一定。还要检查数据库是否有数据、是否按当前 `shopId` 过滤、查询条件是否过窄。
