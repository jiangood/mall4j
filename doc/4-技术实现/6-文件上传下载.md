# 文件上传下载

后台上传接口位于：

```text
yami-shop-admin/src/main/java/com/yami/shop/admin/controller/FileController.java
```

服务实现位于：

```text
yami-shop-service/src/main/java/com/yami/shop/service/impl/AttachFileServiceImpl.java
```

上传配置位于：

```text
yami-shop-common/src/main/resources/shop.properties
```

## 上传入口

当前后台提供：

```text
POST /admin/file/upload/element
POST /admin/file/upload/tinymceEditor
```

管理后台图片组件和富文本组件会调用这些接口。

## 前端组件

当前 `front-end/mall4v/src/components` 下与上传相关的实际组件是：

| 组件 | 用途 | 后端接口 |
| --- | --- | --- |
| `pic-upload` | 单图上传，例如分类图、SKU 图。 | `/admin/file/upload/element` |
| `mul-pic-upload` | 多图上传，例如商品图片，结果为逗号分隔路径。 | `/admin/file/upload/element` |
| `tiny-mce` | 富文本编辑器内插图。 | `/admin/file/upload/element` |

旧文档提到的 `file-upload` 目录在当前管理后台代码中不存在。二开时应以当前组件目录为准。

这些组件都会携带：

```text
Authorization
```

请求头，否则会被后台权限拦截。

## 存储方式

配置项：

```text
shop.imgUpload.uploadType=1  # 本地上传
shop.imgUpload.uploadType=2  # 七牛云
```

本地上传相关配置：

```text
shop.imgUpload.imagePath=D:/Mall4j/IMG/
shop.imgUpload.resourceUrl=http://localhost:8085/mall4j/img/
```

七牛云相关配置：

```text
shop.qiniu.resourcesUrl
shop.qiniu.accessKey
shop.qiniu.secretKey
shop.qiniu.bucket
shop.qiniu.zone
```

前端展示图片时会拼接：

```text
VITE_APP_RESOURCES_URL
```

当前管理后台是 Vite 项目，环境变量前缀是 `VITE_APP_*`。旧文档中的 `VUE_APP_*` 是 Vue CLI 时代的写法，不适用于当前 `front-end/mall4v`。

## 本地资源访问

本地上传的静态资源映射在：

```text
yami-shop-common/src/main/java/com/yami/shop/common/config/ResourceConfigAdapter.java
```

默认将 `/mall4j/img/**` 映射到本地图片目录。

## 二开建议

新增带图片的后台功能时，不要新增上传接口。优先复用现有上传组件和 `/admin/file/upload/element`。
