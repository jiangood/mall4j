# 参数校验

项目使用 Jakarta Validation 做参数校验，常见写法是：

```java
public ServerResponseEntity<Void> save(@Valid @RequestBody XxxParam param)
```

或：

```java
public ServerResponseEntity<Void> save(@Valid XxxParam param)
```

校验失败会进入统一异常处理，最终返回 `ServerResponseEntity.fail(...)`。

## 常见注解

对象字段上常见：

```java
@NotNull
@NotBlank
```

Controller 参数上常见：

```java
@Valid
@RequestBody
```

## 二开建议

新增接口时：

- 必填字段放到参数对象或实体字段上校验。
- Controller 使用 `@Valid` 触发校验。
- 业务规则用 `YamiShopBindException` 表达。
- 不要在前端只做校验，后端必须兜底。

## 示例场景

新增“帮助中心文章”时，标题不能为空可写在参数对象或实体上：

```java
@NotBlank(message = "标题不能为空")
private String title;
```
