# 统一异常处理

统一异常处理位于：

```text
yami-shop-common/src/main/java/com/yami/shop/common/config/DefaultExceptionHandlerConfig.java
```

它使用 `@RestControllerAdvice` 统一拦截接口异常，并返回项目统一响应对象：

```text
ServerResponseEntity
```

## 已处理的异常

| 异常 | 处理方式 |
| --- | --- |
| `MethodArgumentNotValidException` | 处理 `@RequestBody @Valid` 校验失败。 |
| `BindException` | 处理普通参数绑定或校验失败。 |
| `YamiShopBindException` | 处理业务主动抛出的错误。 |
| `NoResourceFoundException` | 返回可展示的失败消息。 |
| `Exception` | 兜底异常，返回通用系统异常。 |

## 二开时怎么用

业务判断失败时，优先抛：

```java
throw new YamiShopBindException("提示信息");
```

不要在 Controller 里随意返回不同结构，也不要直接把异常栈暴露给前端。

## 注意点

当前异常处理大多返回 HTTP `200`，业务成功或失败通过 `ServerResponseEntity` 内部状态表达。前端请求封装也是按这种约定处理的，二开接口应保持一致。
