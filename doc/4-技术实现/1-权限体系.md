# 权限体系

权限体系同时影响后端接口访问、前端菜单显示和前端按钮显示。二开后台功能时必须一起配置。

## 核心数据

菜单和权限数据在表：

```text
tz_sys_menu
```

菜单类型：

| type | 含义 |
| --- | --- |
| `0` | 目录 |
| `1` | 菜单 |
| `2` | 按钮 |

常见字段：

| 字段 | 说明 |
| --- | --- |
| `parent_id` | 上级菜单 ID |
| `name` | 菜单或按钮名称 |
| `url` | 前端路由，菜单类型使用 |
| `perms` | 权限标识，按钮类型常用 |
| `order_num` | 排序 |

## 后端接口权限

后端使用 `@PreAuthorize` 判断权限：

```java
@PreAuthorize("@pms.hasPermission('shop:notice:save')")
```

权限判断实现位于：

```text
yami-shop-security/yami-shop-security-common/src/main/java/com/yami/shop/security/common/permission/PermissionService.java
```

判断逻辑是从当前登录用户上下文中取权限列表，再匹配传入的权限标识。

## 前端菜单权限

登录后，管理后台请求：

```text
GET /sys/menu/nav
```

后端返回：

- `menuList`：当前用户可见菜单。
- `authorities`：当前用户拥有的按钮权限。

前端处理位置：

```text
front-end/mall4v/src/router/index.js
```

当前前端会把按钮权限写入 `sessionStorage` 的 `Authorities`。

前端根据菜单 `url` 动态加载页面：

```text
菜单 url: shop/notice
页面文件: src/views/modules/shop/notice/index.vue
```

## 前端按钮权限

按钮权限判断函数：

```text
front-end/mall4v/src/utils/index.js
```

使用方式：

```js
isAuth('shop:notice:save')
```

如果没有对应权限，按钮不会显示。

## 二开时必须同时做四件事

新增后台功能时，不要只写接口和页面，还要配置权限：

1. `tz_sys_menu` 增加菜单，`type=1`，填写 `url`。
2. `tz_sys_menu` 增加按钮，`type=2`，填写 `perms`。
3. 后端接口使用同一个 `perms` 做 `@PreAuthorize`。
4. 前端按钮使用同一个 `perms` 做 `isAuth()`。

权限标识建议按现有风格：

```text
模块:资源:动作
```

示例：

```text
shop:helpArticle:page
shop:helpArticle:save
shop:helpArticle:update
shop:helpArticle:delete
```
