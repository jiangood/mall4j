# 分页实现

项目分页基于 MyBatis-Plus。

核心文件：

```text
yami-shop-common/src/main/java/com/yami/shop/common/util/PageParam.java
yami-shop-common/src/main/java/com/yami/shop/common/config/MybatisPlusConfig.java
```

`PageParam<T>` 继承 MyBatis-Plus 的 `Page<T>`，默认：

```text
current = 1
size = 10
size 最大限制 = 100
```

## Controller 写法

常见写法：

```java
@GetMapping("/page")
public ServerResponseEntity<IPage<Notice>> page(PageParam<Notice> page, Notice notice) {
    return ServerResponseEntity.success(noticeService.page(page, wrapper));
}
```

前端传参通常是：

```text
current
size
```

## 管理后台分页表格

管理后台列表页常用 AVUE 的 `avue-crud`：

```text
front-end/mall4v/src/views/modules/sys/user/index.vue
front-end/mall4v/src/crud/sys/user.js
```

典型绑定关系：

- `:page` 绑定分页对象。
- `:data` 绑定后端返回的 `records`。
- `:option` 绑定 `tableOption`。
- `@on-load` 触发列表加载。
- `@search-change` 触发查询条件变化。

前端会把 AVUE 的 `currentPage`、`pageSize` 转成后端需要的 `current`、`size`，后端返回后再写回：

```text
data.records -> dataList
data.total -> page.total
```

## MyBatis-Plus 配置

`MybatisPlusConfig` 注册了：

- `PaginationInnerInterceptor`
- `OptimisticLockerInnerInterceptor`

数据库类型配置为 MySQL。

## 复杂关联分页

简单单表或一对一分页优先使用 MyBatis-Plus 自动分页。

一对多、多对多分页不能直接依赖自动 count。例如一个订单有多条订单项时，join 后的行数不是订单数。当前项目在订单明细分页里手写了 records SQL 和 count SQL：

```text
yami-shop-service/src/main/java/com/yami/shop/service/impl/OrderServiceImpl.java
yami-shop-service/src/main/resources/mapper/OrderMapper.xml
```

`OrderServiceImpl.pageOrdersDetailByOrderParam()` 会分别调用：

```text
listOrdersDetailByOrderParam(...)
countOrderDetail(...)
```

这里使用 `PageAdapter` 把 `Page.current`、`Page.size` 转为 SQL 分页参数。当前代码字段是：

```text
begin
size
```

不是旧文档里的 `begin`、`end`。

## 二开建议

新增后台列表接口时，优先沿用：

```text
GET /xxx/page
PageParam<T>
IPage<T>
```

这样能直接适配管理后台现有分页表格。
