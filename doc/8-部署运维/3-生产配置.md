# 生产配置

生产环境不要直接使用本地开发默认配置。

## 必改配置

| 配置 | 位置 |
| --- | --- |
| MySQL 地址、账号、密码 | `application-prod.yml` 或环境变量 |
| Redis 地址和库 | `application-prod.yml` 或环境变量 |
| 文件上传方式 | `shop.properties` |
| 静态资源域名 | `shop.properties` 和前端 `.env.production` |
| 用户端接口域名 | `api.properties` |
| 管理后台接口地址 | `front-end/mall4v/.env.production` |
| 小程序/uni-app 接口地址 | 对应前端环境配置 |
| XXL-JOB 地址 | 管理端配置 |

## 后端生产端口

当前生产配置中：

```text
yami-shop-admin application-prod.yml: 8111
yami-shop-api application-prod.yml: 8112
```

Docker 配置中：

```text
yami-shop-admin application-docker.yml: 8085
yami-shop-api application-docker.yml: 8086
```

部署时要确认 Nginx、前端配置和实际后端端口一致。

## 生产启动

后端打包后，生产环境通常使用 `prod` profile：

```powershell
java -Dspring.profiles.active=prod -jar yami-shop-admin-0.0.1-SNAPSHOT.jar
java -Dspring.profiles.active=prod -jar yami-shop-api-0.0.1-SNAPSHOT.jar
```

注意 JVM 参数和 `-Dspring.profiles.active=prod` 应放在 `-jar` 前面。

## 初始账号

初始化 SQL 中写明后台初始账号为：

```text
admin / 123456
```

生产环境首次登录后应立即修改密码。

## 安全建议

- 不要使用默认数据库密码。
- 不要把对象存储密钥提交到仓库。
- 生产环境必须使用 HTTPS。
- 后台管理域名建议限制访问来源。
- 支付回调必须做验签和幂等。
- 文件上传建议限制大小、类型和访问权限。

## 日志

生产和 Docker 环境使用：

```text
classpath:logback/logback-prod.xml
```

上线前应确认日志目录、滚动策略和磁盘容量。
