# 部署总览

Mall4j 开源版后端部署包含两个 Java 服务：

| 服务 | 说明 | 默认端口 |
| --- | --- | --- |
| `yami-shop-admin` | 管理端接口，管理后台调用。 | 本地/Docker 常用 `8085`，当前 prod 配置为 `8111` |
| `yami-shop-api` | 用户端接口，小程序、H5、uni-app 调用。 | 本地/Docker 常用 `8086`，当前 prod 配置为 `8112` |

依赖服务：

```text
MySQL
Redis
```

可选服务：

```text
XXL-JOB
Nginx
对象存储或本地文件存储
```

## 推荐部署顺序

1. 准备 MySQL，导入 `db/yami_shop.sql`。
2. 准备 Redis。
3. 打包后端。
4. 部署 `yami-shop-admin`。
5. 部署 `yami-shop-api`。
6. 打包并部署管理后台。
7. 配置 Nginx 域名和反向代理。
8. 配置文件上传、支付回调、静态资源域名。

## 后端打包

在仓库根目录执行：

```powershell
mvn clean package -DskipTests
```

生成的 jar 默认位于：

```text
yami-shop-admin/target/yami-shop-admin-0.0.1-SNAPSHOT.jar
yami-shop-api/target/yami-shop-api-0.0.1-SNAPSHOT.jar
```

## 前端打包

管理后台：

```powershell
cd front-end\mall4v
pnpm i
pnpm run build
```

构建输出目录以 Vite 配置为准。

## 完成后应该看到

- 两个后端 jar 已生成。
- 生产配置中的 MySQL、Redis 地址已改成真实环境。
- 管理后台静态文件已构建完成。
- Nginx 能访问前端页面并转发接口。
- 管理后台能成功登录。

## 检查题-含答案

问题：为什么部署时要同时考虑 `yami-shop-admin` 和 `yami-shop-api`？

答案：管理后台依赖 `yami-shop-admin`，用户端、小程序、H5、uni-app 依赖 `yami-shop-api`。只部署管理后台可以先不启动用户端，但完整商城需要两个服务。

问题：生产端口为什么不能只看 README 的本地端口？

答案：不同 profile 端口不同。当前 `dev/docker` 和 `prod` 配置不一样，上线要以实际启用的 profile 配置为准。
