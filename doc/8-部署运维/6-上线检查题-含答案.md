# 上线检查题-含答案

这些问题用于上线前自查。每题都给答案，方便你边检查边修。

## 后端

问题：生产环境应该启用哪个 profile？

答案：jar 部署通常启用 `prod`：`-Dspring.profiles.active=prod`。

问题：`-Dspring.profiles.active=prod` 应该放在 `-jar` 前还是后？

答案：放在 `-jar` 前。JVM 参数放在 `-jar` 后可能被当作应用参数。

问题：当前生产配置里管理端和用户端端口是多少？

答案：当前 `application-prod.yml` 中，管理端是 `8111`，用户端是 `8112`。

## 数据库和 Redis

问题：生产 MySQL 只改管理端配置够吗？

答案：不够。`yami-shop-admin` 和 `yami-shop-api` 各有自己的 `application-prod.yml`，都要确认数据库和 Redis 配置。

问题：初始化 SQL 导入哪个数据库？

答案：导入 `yami_shops`。

## 前端

问题：管理后台生产构建命令是什么？

答案：

```powershell
cd front-end\mall4v
pnpm i
pnpm run build
```

问题：生产接口地址在哪里配置？

答案：管理后台在 `front-end/mall4v/.env.production` 配置 `VITE_APP_BASE_API`。

问题：当前管理后台环境变量前缀是 `VUE_APP_` 吗？

答案：不是。当前是 Vite 项目，应使用 `VITE_APP_*`。

## Nginx

问题：Nginx 在这里主要做什么？

答案：托管管理后台静态文件，并把接口路径反向代理到后端服务。

问题：如果 `VITE_APP_BASE_API='/apis'`，还需要做什么？

答案：Nginx 必须配置 `/apis` 的 rewrite 和 `proxy_pass`，否则前端请求找不到后端。

## 文件上传和图片

问题：图片上传成功但生产页面不显示，先查什么？

答案：先查 `VITE_APP_RESOURCES_URL`、后端文件上传配置、静态资源域名或本地资源映射是否可访问。

问题：本地上传和七牛云上传由哪个配置决定？

答案：`shop.imgUpload.uploadType`。`1` 是本地上传，`2` 是七牛云。

## Docker

问题：当前仓库能直接 `docker compose up -d` 吗？

答案：不能保证。当前 compose 引用了 `db/Dockerfile`，但 `db/` 当前只有 `yami_shop.sql`，需要先修正 MySQL 初始化方式。

问题：Docker 环境下数据库连接配置来自哪里？

答案：来自 `application-docker.yml` 和 Docker 环境变量，例如 `MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_DATABASE`。

## 安全

问题：生产环境可以继续使用 `admin / 123456` 吗？

答案：不应该。首次登录后必须修改默认密码。

问题：支付回调只改前端按钮够吗？

答案：不够。真实支付必须补齐服务端回调、验签、幂等和异常状态处理。
