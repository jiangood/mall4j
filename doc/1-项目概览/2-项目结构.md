# 项目结构

仓库根目录主要分为后端、前端、数据库、文档和部署配置。

```text
mall4j/
  db/                         # 数据库初始化脚本，当前只有 yami_shop.sql
  doc/                        # 面向维护者和二开的技术文档
  docs/agents/                # Agent 工作规范，不是业务技术文档入口
  front-end/
    mall4v/                   # Vue3 管理后台
    mall4m/                   # 原生微信小程序端
    mall4uni/                 # uni-app 端
  yami-shop-admin/            # 管理端后端接口
  yami-shop-api/              # 用户端后端接口
  yami-shop-bean/             # 实体、DTO、VO 等对象
  yami-shop-common/           # 通用工具、响应、异常、注解
  yami-shop-security/         # 登录认证和权限相关模块
  yami-shop-service/          # 业务服务、Mapper、持久层
  yami-shop-sys/              # 系统管理：菜单、角色、日志、配置
  docker-compose.yml          # 本地 Docker 编排配置
  pom.xml                     # Maven 父工程
```

## 接手时先看哪里

1. 看 `pom.xml`，确认后端模块和依赖版本。
2. 看 `yami-shop-admin` 和 `yami-shop-api`，确认两个后端入口。
3. 看 `front-end/mall4v`，确认管理后台页面和动态路由。
4. 看 `db/yami_shop.sql`，确认初始化表结构和菜单权限数据。
5. 看 `doc/`，按导航补齐系统理解。
