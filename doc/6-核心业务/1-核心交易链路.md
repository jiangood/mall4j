# 核心交易链路

核心交易链路是：

```text
商品/SKU -> 加入购物车 -> 确认订单 -> 提交订单 -> 支付 -> 订单管理
```

## 第一阶段理解重点

先分清两类接口：

| 端 | 模块 | 说明 |
| --- | --- | --- |
| 用户端 | `yami-shop-api` | 浏览商品、购物车、下单、支付。 |
| 管理端 | `yami-shop-admin` | 管理商品、订单、发货、配置。 |

## 代码入口

常见入口：

```text
yami-shop-api/src/main/java/com/yami/shop/api/controller/ShopCartController.java
yami-shop-api/src/main/java/com/yami/shop/api/controller/OrderController.java
yami-shop-api/src/main/java/com/yami/shop/api/controller/PayController.java
yami-shop-admin/src/main/java/com/yami/shop/admin/controller/ProductController.java
yami-shop-admin/src/main/java/com/yami/shop/admin/controller/OrderController.java
```

## 主链路接口

| 环节 | 接口入口 | 说明 |
| --- | --- | --- |
| 商品浏览 | `ProdController`、`SkuController` | 用户端读取商品和 SKU。 |
| 购物车 | `ShopCartController` | 添加、修改、删除、统计购物车商品。 |
| 确认订单 | `OrderController.confirm` | 生成确认订单快照，并写入确认订单缓存。 |
| 提交订单 | `OrderController.submitOrders` | 从确认订单缓存创建订单和订单项。 |
| 支付 | `PayController` | 生成支付流水并更新支付状态。 |
| 后台订单 | 管理端 `OrderController` | 订单查询、详情、发货、导出。 |

## 关键设计

- 用户端接口集中在 `yami-shop-api`。
- 管理端接口集中在 `yami-shop-admin`。
- 订单确认结果会先放入 `ConfirmOrderCache`，提交订单时再读取。
- 提交订单后会清理确认订单缓存，并在购物车下单时清理购物车缓存。
- 支付成功会发布支付成功事件，后续逻辑通过事件扩展。

## 不建议新手先改的地方

- 库存扣减和返还。
- 支付状态流转。
- 订单状态流转。
- 订单自动取消、自动确认收货任务。

这些地方影响交易一致性，改动前应先补测试或至少完整跑通交易链路。
