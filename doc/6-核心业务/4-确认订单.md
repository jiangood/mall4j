# 确认订单

确认订单不是最终下单，而是把用户选择的商品、地址、金额等信息组装成一个确认订单快照。

入口：

```text
POST /p/order/confirm
```

代码位置：

```text
yami-shop-api/src/main/java/com/yami/shop/api/controller/OrderController.java
yami-shop-api/src/main/java/com/yami/shop/api/listener/ConfirmOrderListener.java
yami-shop-bean/src/main/java/com/yami/shop/bean/app/param/OrderParam.java
```

`OrderParam` 同时支持两种入口：

| 字段 | 含义 |
| --- | --- |
| `basketIds` | 购物车结算。 |
| `orderItem` | 立即购买。 |
| `addrId` | 收货地址 ID，必填。 |
| `couponIds` | 优惠券选择，当前开源版保留参数。 |

`BasketServiceImpl.getShopCartItemsByOrderItems()` 会把购物车结算和立即购买统一转换成购物项列表。立即购买不会先写入购物车，购物车结算也不会在确认订单阶段删除购物车记录。

## 主要流程

1. 读取当前登录用户。
2. 根据地址 ID 查询用户收货地址。
3. 根据购物车 ID 或立即购买参数组装商品项。
4. 按店铺聚合购物车商品。
5. 发布 `ConfirmOrderEvent`，让扩展逻辑补充订单金额、运费等信息。
6. 汇总总金额、实际金额、商品数量、优惠金额。
7. 写入 `ConfirmOrderCache`。
8. 返回确认订单信息给前端。

## 默认金额计算

当前开源版默认监听器 `ConfirmOrderListener` 会：

- 校验商品和 SKU 是否存在、是否上架。
- 统计商品总数和商品总额。
- 有收货地址时，通过 `TransportManagerService.calculateTransfee()` 计算运费。
- 将店铺订单实际金额设置为 `商品总额 + 运费`。

旧文档里有一处把实际金额写成 `total - transfee` 的片段，已不符合当前代码；当前代码是 `Arith.add(total, transfee)`。

## 确认订单缓存

确认订单结果通过：

```text
OrderService.putConfirmOrderCache(userId, data)
```

写入缓存：

```text
ConfirmOrderCache
```

提交订单时会再从缓存读取。如果缓存不存在，会提示：

```text
订单已过期，请重新下单
```

## 二开注意

确认订单阶段适合做金额展示、运费计算、优惠计算等“预计算”。真正创建订单在提交订单阶段完成。
