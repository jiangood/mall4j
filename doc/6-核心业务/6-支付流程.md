# 支付流程

支付入口：

```text
POST /p/order/pay
POST /p/order/normalPay
```

代码位置：

```text
yami-shop-api/src/main/java/com/yami/shop/api/controller/PayController.java
yami-shop-service/src/main/java/com/yami/shop/service/impl/PayServiceImpl.java
```

## 当前实现

当前代码会：

1. 根据订单号生成统一支付流水号 `payNo`。
2. 更新订单结算表中的支付流水、支付方式、用户和订单号。
3. 汇总同一支付流水下的应付金额。
4. 调用 `paySuccess()` 更新支付状态。
5. 将订单更新为已支付。
6. 发布 `PaySuccessOrderEvent`。

也就是说，当前开源实现里支付接口会直接调用支付成功逻辑，适合本地跑通交易流程。接入真实第三方支付时，需要重新处理支付网关、回调验签、幂等和异常状态。

## 支付参数

支付参数只包含订单号集合和支付方式：

```text
yami-shop-bean/src/main/java/com/yami/shop/bean/app/param/PayParam.java
```

当前接口不接受前端传入订单金额。`PayServiceImpl` 会按 `orderNumbers` 拆分订单号，读取 `tz_order_settlement` 中同一 `payNo` 下的 `pay_amount` 汇总应付金额。

多个订单号可以共用一个 `payNo`，因此支持多店铺订单的并单支付；单个订单号则是普通单订单支付。

## 涉及表

```text
tz_order
tz_order_settlement
```

## 支付回调

用户端存在支付回调 Controller：

```text
yami-shop-api/src/main/java/com/yami/shop/api/controller/PayNoticeController.java
```

当前其中支付回调逻辑是注释状态。真实支付接入不能只改前端按钮，需要补齐服务端回调。

## 二开注意

支付属于高风险模块。改动前必须明确：

- 支付流水号和订单号关系。
- 重复回调如何处理。
- 已支付订单再次支付如何处理。
- 支付成功后订单状态和结算状态如何保持一致。
