# 购物车

购物车用户端入口：

```text
yami-shop-api/src/main/java/com/yami/shop/api/controller/ShopCartController.java
```

核心表：

```text
tz_basket
```

购物车不是只返回 `tz_basket` 原始记录，而是会关联商品、SKU、店铺和用户信息后，按店铺组装成前端需要的结构。

## 主要接口

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/p/shopCart/info` | `POST` | 获取购物车信息，可同步前端选中项参数。 |
| `/p/shopCart/changeItem` | `POST` | 添加或修改购物车商品数量。 |
| `/p/shopCart/deleteItem` | `DELETE` | 删除指定购物车项。 |
| `/p/shopCart/deleteAll` | `DELETE` | 清空购物车。 |
| `/p/shopCart/prodCount` | `GET` | 获取购物车商品数量。 |
| `/p/shopCart/expiryProdList` | `GET` | 获取失效商品。 |
| `/p/shopCart/cleanExpiryProdList` | `DELETE` | 清空失效商品。 |
| `/p/shopCart/totalPay` | `POST` | 统计选中商品金额和数量。 |

## 添加商品规则

`changeItem` 使用 `count` 表示变化数量：

- `count > 0`：增加数量。
- `count < 0`：减少数量。
- 最终数量小于等于 0 时删除购物车项。

添加或修改前会校验：

- 商品状态是否正常。
- SKU 状态是否正常。
- SKU 库存是否足够。

## 唯一约束

`tz_basket` 有唯一索引：

```text
uk_user_shop_sku(sku_id, user_id, shop_id)
```

同一用户、同一店铺、同一 SKU 只能有一条购物车记录。

## 扩展点

购物车聚合逻辑位于：

```text
yami-shop-service/src/main/java/com/yami/shop/service/impl/BasketServiceImpl.java
```

`getShopCarts()` 会先按 `shopId` 把购物项分组，再发布：

```text
ShopCartEvent
```

默认监听器：

```text
yami-shop-api/src/main/java/com/yami/shop/api/listener/ShopCartListener.java
```

当前开源版默认监听器只是把店铺下的商品组装到一个 `ShopCartItemDiscountDto` 中。旧文档里提到的满减、满折等“装饰”逻辑是扩展思路，当前开源版没有完整营销实现。二开优惠活动时，可以沿着 `ShopCartEvent` 增加监听器，而不是把所有计算塞进 Controller。
