# 提交订单

提交订单会把确认订单快照写入正式订单表。

入口：

```text
POST /p/order/submit
```

代码位置：

```text
yami-shop-api/src/main/java/com/yami/shop/api/controller/OrderController.java
yami-shop-api/src/main/java/com/yami/shop/api/listener/SubmitOrderListener.java
yami-shop-service/src/main/java/com/yami/shop/service/impl/OrderServiceImpl.java
```

## 主要流程

1. 读取当前登录用户。
2. 从 `ConfirmOrderCache` 获取确认订单快照。
3. 将前端提交的店铺备注合并到对应店铺订单。
4. 调用 `orderService.submit()` 创建订单。
5. 通过 `SubmitOrderEvent` 组装订单和订单项。
6. 批量保存订单。
7. 批量保存订单项。
8. 清理商品和 SKU 缓存。
9. 如果来自购物车，清理用户购物车缓存。
10. 清理确认订单缓存。
11. 返回订单号集合。

## 关键设计

提交订单阶段有几个容易漏掉的点：

- 地址保存为 `UserAddrOrder` 快照，而不是只保存用户地址 ID。这样用户后续修改收货地址，不会影响已提交订单。
- 每个店铺生成一个独立订单，便于商家结算、发货和确认收货。
- 订单号使用 Snowflake 生成，不直接使用数据库主键。
- 如果来自购物车，下单成功后删除已提交的购物车项。
- 订单结算金额写入 `tz_order_settlement.pay_amount`，后续支付按数据库金额计算。

## 库存扣减

库存扣减使用数据库条件更新防止超卖：

```text
yami-shop-service/src/main/resources/mapper/SkuMapper.xml
yami-shop-service/src/main/resources/mapper/ProductMapper.xml
```

核心条件是：

```sql
where sku_id = ? and ? <= stocks
where prod_id = ? and ? <= total_stocks
```

更新失败时会清理对应商品或 SKU 缓存，并抛出库存不足异常。

## 涉及表

```text
tz_order
tz_order_item
tz_order_settlement
tz_user_addr_order
```

## 二开注意

不要绕过确认订单直接创建订单。当前设计依赖确认订单缓存承接前一步的金额、地址、商品项和店铺拆单结果。
