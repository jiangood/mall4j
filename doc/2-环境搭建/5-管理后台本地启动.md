# 管理后台本地启动

管理后台位于：

```text
front-end/mall4v
```

## 启动步骤

PowerShell 示例：

```powershell
cd front-end\mall4v
pnpm i
pnpm run dev
```

启动后按终端输出访问本地地址。

## 接口地址

开发环境接口地址在：

```text
front-end/mall4v/.env.development
```

默认配置：

```text
VITE_APP_BASE_API = 'http://127.0.0.1:8085'
VITE_APP_RESOURCES_URL = 'https://img.mall4j.com/'
```

管理后台必须连接 `yami-shop-admin`，也就是默认 `8085`。

如果后端端口改了，前端也要同步修改 `VITE_APP_BASE_API`。

## 完成后应该看到

- 管理后台页面能打开。
- 浏览器 Network 中接口请求发往 `http://127.0.0.1:8085`。
- 使用 `admin / 123456` 能登录。
- 登录后能看到左侧菜单。

## 检查题-含答案

问题：管理后台前端端口和后端端口为什么不同？

答案：项目是前后端分离。前端由 Vite 启动，当前端口 `9527`；后端管理接口是 Spring Boot 服务，默认端口 `8085`。

问题：如果登录请求发往 `8086`，说明什么？

答案：说明管理后台接口地址配错了。管理后台必须连 `yami-shop-admin` 的 `8085`。
