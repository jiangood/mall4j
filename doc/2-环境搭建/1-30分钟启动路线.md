# 30 分钟启动路线

这篇只解决一个目标：先把管理后台跑起来。不要一开始就同时启动小程序、uni-app、Docker、Nginx、支付回调。

## 最小闭环

本地第一次启动只需要：

```text
MySQL -> Redis -> yami-shop-admin -> front-end/mall4v
```

对应端口：

| 服务 | 端口 | 用途 |
| --- | --- | --- |
| MySQL | `3306` | 存储业务数据和菜单权限。 |
| Redis | `6379` | 缓存、登录等依赖。 |
| `yami-shop-admin` | `8085` | 管理后台接口。 |
| `front-end/mall4v` | `9527` | 管理后台页面。 |

## 启动顺序

1. 确认 JDK、Maven、MySQL、Redis、Node.js、pnpm 版本。
2. 启动 MySQL。
3. 导入 `db/yami_shop.sql`。
4. 启动 Redis。
5. 启动 `yami-shop-admin`。
6. 启动 `front-end/mall4v`。
7. 打开管理后台页面，使用初始化账号登录。

初始化账号：

```text
admin / 123456
```

## 先不要做的事

第一次启动不要先做这些：

- 不要先改 Docker。
- 不要先配 Nginx。
- 不要先接支付。
- 不要先启动小程序和 uni-app。
- 不要先写二开功能。

先跑通管理后台，后面的理解成本会低很多。

## 完成后应该看到

- `http://127.0.0.1:8085/doc.html` 能打开管理端接口文档。
- 管理后台本地页面能打开。
- 登录接口请求的是 `http://127.0.0.1:8085`。
- 登录成功后能看到左侧菜单。

## 检查题-含答案

问题：第一次启动为什么只跑 `yami-shop-admin`，不一定要跑 `yami-shop-api`？

答案：管理后台只依赖管理端接口 `8085`。先跑通后台管理闭环，可以最快确认数据库、Redis、登录、菜单权限、前端接口地址都正常。

问题：管理后台页面端口是 `8085` 吗？

答案：不是。`8085` 是管理端后端接口；管理后台前端由 Vite 启动，当前端口是 `9527`。

问题：登录失败时应该先怀疑业务代码吗？

答案：不应该。先查 MySQL 是否导入数据、Redis 是否启动、管理后台是否连 `8085`、Token 是否过期。
