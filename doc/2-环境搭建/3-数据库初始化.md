# 数据库初始化

初始化脚本位于：

```text
db/yami_shop.sql
```

本地配置默认连接数据库：

```text
数据库名: yami_shops
账号: root
密码: root
端口: 3306
```

对应配置文件：

```text
yami-shop-admin/src/main/resources/application-dev.yml
yami-shop-api/src/main/resources/application-dev.yml
```

## 初始化步骤

1. 启动 MySQL。
2. 创建数据库。
3. 导入 `db/yami_shop.sql`。
4. 确认 `application-dev.yml` 中的数据库地址、账号、密码和本地一致。

PowerShell 示例：

```powershell
mysql -uroot -p
```

进入 MySQL 后执行：

```sql
CREATE DATABASE IF NOT EXISTS yami_shops DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
USE yami_shops;
SOURCE D:/workspace/gitee/mall4j/db/yami_shop.sql;
```

如果本地仓库路径不同，替换 `SOURCE` 后面的路径。

## 完成后应该看到

- 数据库中存在 `yami_shops`。
- 能看到 `tz_sys_user`、`tz_sys_menu` 等表。
- `tz_sys_user` 中有初始化管理员。
- `tz_sys_menu` 中有后台菜单和按钮权限数据。

## 检查题-含答案

问题：为什么启动前必须先导入 `db/yami_shop.sql`？

答案：后端启动和后台登录依赖数据库表、初始账号、菜单权限等初始化数据。没有导入 SQL，即使服务启动了也无法正常登录和显示菜单。

问题：数据库账号密码在哪里改？

答案：本地开发改两个 `application-dev.yml`：管理端和用户端各有一份。
