# 后端本地启动

本地后端有两个启动入口。

| 服务 | 启动类 | 默认端口 | 用途 |
| --- | --- | --- | --- |
| 管理端接口 | `WebApplication` | `8085` | 管理后台调用 |
| 用户端接口 | `ApiApplication` | `8086` | 小程序、H5、uni-app 调用 |

## 启动前检查

先确认：

- MySQL 已启动，且已导入 `db/yami_shop.sql`。
- Redis 已启动，端口为 `6379`。
- `application-dev.yml` 中数据库账号密码正确。
- JDK 为 `17`。

## 用 IDE 启动

推荐用 IDEA 导入 Maven 项目，然后分别运行：

```text
yami-shop-admin/src/main/java/com/yami/shop/admin/WebApplication.java
yami-shop-api/src/main/java/com/yami/shop/api/ApiApplication.java
```

只跑管理后台时，先启动 `WebApplication` 即可。

## 用 Maven 启动

在仓库根目录执行：

```powershell
mvn -pl yami-shop-admin -am spring-boot:run
```

用户端接口需要另开一个终端：

```powershell
mvn -pl yami-shop-api -am spring-boot:run
```

## 接口文档

管理端后端启动后，可访问：

```text
http://127.0.0.1:8085/doc.html
```

用户端后端启动后，可访问：

```text
http://127.0.0.1:8086/doc.html
```

## 完成后应该看到

- 管理端接口文档 `http://127.0.0.1:8085/doc.html` 能打开。
- 控制台没有数据库连接失败。
- 控制台没有 Redis 连接失败。
- `8085` 端口没有被其他进程占用。

## 检查题-含答案

问题：管理后台本地启动前，必须先启动哪个 Java 服务？

答案：必须先启动 `yami-shop-admin`，也就是 `WebApplication`，默认端口 `8085`。

问题：`yami-shop-api` 不启动，管理后台能不能先跑？

答案：可以。管理后台调用的是 `yami-shop-admin`，用户端接口可以等后台跑通后再启动。
