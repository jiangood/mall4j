# 后端分层

后端按职责分层，二开时尽量沿用现有放置规则。

## Controller

位置：

```text
yami-shop-admin/src/main/java/com/yami/shop/admin/controller/
yami-shop-api/src/main/java/com/yami/shop/api/controller/
```

职责：

- 接收 HTTP 请求。
- 做权限注解。
- 做简单参数组织。
- 调用 Service。
- 返回 `ServerResponseEntity`。

## Service

位置：

```text
yami-shop-service/src/main/java/com/yami/shop/service/
```

职责：

- 承载业务规则。
- 组合多个 Mapper。
- 处理缓存、状态变化、事务等逻辑。

## Mapper

位置：

```text
yami-shop-service/src/main/java/com/yami/shop/dao/
yami-shop-service/src/main/resources/mapper/
```

职责：

- 访问数据库。
- 使用 MyBatis / MyBatis-Plus 完成查询和写入。

## Model、DTO、VO

位置：

```text
yami-shop-bean/src/main/java/com/yami/shop/bean/
```

常见用途：

- `model`：数据库实体。
- `dto`：请求或业务传输对象。
- `vo`：返回给前端的展示对象。

## 通用返回

接口通常返回：

```text
ServerResponseEntity<T>
```

二开新增接口时，优先跟随现有 Controller 的返回风格。
