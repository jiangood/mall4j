# 数据库表阅读路线

这篇只解决一个目标：让你知道第一次看 `db/yami_shop.sql` 应该先看哪些表。

## 先看三类表

第一次不要从 SQL 第一行看到最后一行。先看这三类：

| 优先级 | 表 | 为什么先看 |
| --- | --- | --- |
| 1 | `tz_sys_user`、`tz_sys_menu`、`tz_sys_role`、`tz_sys_role_menu`、`tz_sys_user_role` | 决定后台能不能登录、菜单和按钮能不能显示。 |
| 2 | `tz_prod`、`tz_sku`、`tz_basket` | 决定商品、SKU、购物车如何进入交易链路。 |
| 3 | `tz_order`、`tz_order_item`、`tz_order_settlement`、`tz_user_addr_order` | 决定下单后订单、订单项、支付金额、地址快照如何保存。 |

## 启动失败时先看权限表

后台能启动但登录后菜单异常时，先看：

```text
tz_sys_user
tz_sys_user_role
tz_sys_role
tz_sys_role_menu
tz_sys_menu
```

这几张表决定：

- 默认账号是否存在。
- 用户拥有哪些角色。
- 角色拥有哪些菜单。
- 菜单 URL 如何映射前端页面。
- 按钮权限如何映射后端接口和前端按钮。

## 理解交易时再看商品和订单

交易链路建议这样读：

```text
商品 tz_prod
  -> SKU tz_sku
  -> 购物车 tz_basket
  -> 订单 tz_order
  -> 订单项 tz_order_item
  -> 订单结算 tz_order_settlement
  -> 订单地址快照 tz_user_addr_order
```

## 二开时先看相似表

新增一个后台 CRUD 时，不要直接设计全新风格。先找相似表：

| 你要做的功能 | 可参考 |
| --- | --- |
| 帮助文章、公告、说明类内容 | `tz_notice` |
| 热搜、轮播、配置类内容 | `tz_hot_search`、`tz_index_img`、`tz_sys_config` |
| 商品标签、分组类内容 | `tz_prod_tag`、`tz_prod_tag_reference` |
| 带店铺隔离的数据 | 看是否有 `shop_id` 字段 |

## 完成后应该知道

- 后台登录和菜单异常先查权限表。
- 商品下单最终落到 SKU。
- 一个订单会拆出订单项和结算记录。
- 订单地址保存快照，不只关联用户当前地址。
- 二开新表要先找现有相似表。

## 检查题-含答案

问题：后台登录成功但左侧菜单不对，先看商品表还是权限表？

答案：先看权限表，尤其是 `tz_sys_menu`、`tz_sys_role_menu`、`tz_sys_user_role`。

问题：购物车和下单最终依赖商品表还是 SKU 表？

答案：最终依赖 SKU。商品用于展示和聚合，具体购买、价格、库存会落到 `tz_sku`。

问题：为什么订单地址要保存到 `tz_user_addr_order`？

答案：这是订单地址快照。用户下单后即使修改收货地址，也不能影响历史订单的收货信息。
