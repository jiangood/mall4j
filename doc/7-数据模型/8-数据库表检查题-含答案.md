# 数据库表检查题-含答案

这些问题用于确认你是否能从数据库角度读懂系统。每题都给答案。

## 初始化

问题：当前表结构以哪里为准？

答案：以 `db/yami_shop.sql` 为准。旧文档和截图只能作为参考。

问题：默认后台账号在哪张表？

答案：在 `tz_sys_user`。

问题：默认菜单和按钮权限在哪张表？

答案：在 `tz_sys_menu`。

## 权限

问题：后台用户和角色是什么关系？

答案：`tz_sys_user` 通过 `tz_sys_user_role` 关联 `tz_sys_role`。

问题：角色和菜单是什么关系？

答案：`tz_sys_role` 通过 `tz_sys_role_menu` 关联 `tz_sys_menu`。

问题：菜单、页面、按钮权限都存在一张表里吗？

答案：是。都在 `tz_sys_menu`，通过 `type` 区分目录、菜单、按钮。

问题：`tz_sys_menu.type=2` 表示什么？

答案：表示按钮权限，例如新增、修改、删除。

## 商品

问题：商品主表是哪张？

答案：`tz_prod`。

问题：SKU 表是哪张？

答案：`tz_sku`。

问题：商品和 SKU 如何关联？

答案：`tz_prod.prod_id = tz_sku.prod_id`。

问题：购物车记录具体商品时更关注商品还是 SKU？

答案：更关注 SKU，因为具体购买规格、价格、库存都落到 SKU。

## 订单

问题：订单主表是哪张？

答案：`tz_order`。

问题：订单项表是哪张？

答案：`tz_order_item`。

问题：订单和订单项通过什么字段关联？

答案：通过 `order_number` 关联。

问题：支付流水和支付金额主要看哪张表？

答案：看 `tz_order_settlement`。

问题：订单为什么要保存地址快照？

答案：为了保证历史订单地址不受用户后续修改收货地址影响，地址快照存到 `tz_user_addr_order`。

## 二开

问题：新增后台 CRUD 表时，为什么经常需要 `shop_id`？

答案：因为后台数据通常要按店铺隔离，避免一个店铺操作到另一个店铺的数据。

问题：新增菜单后，为什么数据库有数据但页面还是不显示？

答案：可能没有给当前角色分配菜单权限，或菜单 `url` 和前端页面路径不一致。

问题：数据库表文档写错时，应该信文档还是信 SQL？

答案：信当前 SQL 和当前代码。文档必须回到 `db/yami_shop.sql` 和源码交叉验证。

## 使用建议

读完后建议打开 `db/yami_shop.sql`，按“权限 -> 商品 -> 订单”的顺序各找一遍表结构和初始数据。
