# 用户与权限数据模型

项目有两类用户：

- 商城用户：下单、收货、收藏商品。
- 后台用户：登录管理后台、管理商品和订单。

## 商城用户

核心表：

```text
tz_user
tz_user_addr
tz_user_addr_order
tz_user_collection
```

`tz_user` 记录用户基础信息，例如昵称、手机号、头像、状态、积分。

`tz_user_addr` 记录用户收货地址。

`tz_user_addr_order` 记录订单地址快照，用于保存下单当时的地址信息。

`tz_user_collection` 记录用户收藏。

## 后台用户与权限

核心表：

```text
tz_sys_user
tz_sys_user_role
tz_sys_role
tz_sys_role_menu
tz_sys_menu
```

关系：

```text
tz_sys_user
  -> tz_sys_user_role
  -> tz_sys_role
  -> tz_sys_role_menu
  -> tz_sys_menu
```

## 菜单权限

`tz_sys_menu.type`：

| 值 | 含义 |
| --- | --- |
| `0` | 目录 |
| `1` | 菜单 |
| `2` | 按钮 |

`url` 用于前端动态路由，`perms` 用于后端接口和前端按钮权限。

## 初始后台账号

初始化 SQL 中有默认后台账号：

```text
admin / 123456
```

如果登录失败，先确认 `tz_sys_user`、`tz_sys_role`、`tz_sys_role_menu`、`tz_sys_menu` 是否导入完整。

## 完成后应该知道

- 商城用户和后台用户不是同一套表。
- 后台权限由用户、角色、角色菜单、菜单权限共同决定。
- 菜单 `url` 决定前端动态路由。
- 按钮 `perms` 同时影响后端接口权限和前端按钮显示。

## 检查题-含答案

问题：商城用户登录和后台管理员登录看同一张用户表吗？

答案：不是。商城用户看 `tz_user`，后台管理员看 `tz_sys_user`。

问题：按钮权限字段叫什么？

答案：`tz_sys_menu.perms`。

问题：菜单 URL 字段有什么用？

答案：`url` 用于管理后台动态路由，前端会按它加载 `front-end/mall4v/src/views/modules/{url}/index.vue`。
