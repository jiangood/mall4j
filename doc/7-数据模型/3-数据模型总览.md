# 数据模型总览

数据库初始化脚本：

```text
db/yami_shop.sql
```

二开时优先通过初始化脚本确认当前表结构，不要只看旧截图或旧说明。

## 先记住一句话

数据库表不是孤立看的。启动和二开最常用的三条线是：

- 后台能不能登录和显示菜单：看权限表。
- 商品能不能加入购物车和下单：看商品、SKU、购物车表。
- 订单金额、支付、地址是否正确：看订单、订单项、结算、地址快照表。

## 重点表

| 主题 | 典型表 |
| --- | --- |
| 商品 | `tz_prod`、`tz_sku`、`tz_prod_prop`、`tz_prod_prop_value`、`tz_prod_tag`、`tz_prod_tag_reference` |
| 购物车 | `tz_basket` |
| 订单 | `tz_order`、`tz_order_item`、`tz_order_settlement`、`tz_order_refund` |
| 用户 | `tz_user`、`tz_user_addr`、`tz_user_addr_order`、`tz_user_collection` |
| 权限 | `tz_sys_menu`、`tz_sys_role`、`tz_sys_role_menu`、`tz_sys_user`、`tz_sys_user_role` |

## 读表顺序

理解交易链路时，建议按这个顺序看表：

```text
tz_prod / tz_sku
  -> tz_basket
  -> tz_order / tz_order_item / tz_order_settlement
  -> tz_user / tz_user_addr / tz_user_addr_order
```

理解后台权限时，建议按这个顺序看表：

```text
tz_sys_user
  -> tz_sys_user_role
  -> tz_sys_role
  -> tz_sys_role_menu
  -> tz_sys_menu
```

## 完成后应该知道

- `db/yami_shop.sql` 是当前表结构基准。
- 后台权限不是只靠前端控制，而是由权限表、后端注解、前端按钮共同决定。
- 商品和 SKU 是一对多关系。
- 订单主表之外，还要看订单项、结算表和地址快照。

## 检查题-含答案

问题：为什么不建议只看旧截图理解数据库？

答案：截图可能来自旧版本 UI 或旧表结构。当前表结构应以 `db/yami_shop.sql` 和当前代码为准。

问题：后台菜单不显示，最先查哪组表？

答案：查权限表：`tz_sys_user`、`tz_sys_user_role`、`tz_sys_role`、`tz_sys_role_menu`、`tz_sys_menu`。
