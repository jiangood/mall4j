# 核心表速查

这张表用于快速定位“某个功能大概看哪张表”。

| 模块 | 表 | 用途 |
| --- | --- | --- |
| 后台用户 | `tz_sys_user` | 后台管理员账号。 |
| 后台角色 | `tz_sys_role` | 后台角色。 |
| 用户角色关系 | `tz_sys_user_role` | 用户拥有哪些角色。 |
| 菜单按钮 | `tz_sys_menu` | 后台目录、菜单、按钮权限。 |
| 角色菜单关系 | `tz_sys_role_menu` | 角色拥有哪些菜单和按钮权限。 |
| 商城用户 | `tz_user` | 用户端会员。 |
| 用户地址 | `tz_user_addr` | 当前收货地址。 |
| 订单地址快照 | `tz_user_addr_order` | 下单时复制出来的地址。 |
| 商品 | `tz_prod` | 商品主信息。 |
| SKU | `tz_sku` | 具体可购买规格、价格、库存。 |
| 商品规格 | `tz_prod_prop`、`tz_prod_prop_value` | 规格名和规格值。 |
| 商品分组 | `tz_prod_tag`、`tz_prod_tag_reference` | 商品标签和商品分组关联。 |
| 购物车 | `tz_basket` | 用户购物车项。 |
| 订单 | `tz_order` | 店铺维度订单主记录。 |
| 订单项 | `tz_order_item` | 订单中的具体商品项。 |
| 订单结算 | `tz_order_settlement` | 支付流水、支付金额、支付状态。 |
| 退款 | `tz_order_refund` | 退款记录。 |
| 附件 | `tz_attach_file` | 上传文件记录。 |
| 店铺 | `tz_shop_detail` | 店铺基础信息。 |
| 分类 | `tz_category` | 商品分类。 |

## 最常查的三组关系

权限：

```text
tz_sys_user -> tz_sys_user_role -> tz_sys_role -> tz_sys_role_menu -> tz_sys_menu
```

商品：

```text
tz_prod -> tz_sku
tz_prod -> tz_prod_tag_reference -> tz_prod_tag
```

订单：

```text
tz_order.order_number = tz_order_item.order_number
tz_order.order_number = tz_order_settlement.order_number
tz_order.addr_order_id = tz_user_addr_order.addr_order_id
```

## 检查题-含答案

问题：按钮权限存在哪张表？

答案：存在 `tz_sys_menu`。按钮权限是 `type=2` 的菜单记录，权限标识写在 `perms` 字段。

问题：支付金额主要看订单主表还是订单结算表？

答案：看 `tz_order_settlement`。支付流程会从结算表读取应付金额并更新支付流水。

问题：商品分组为什么需要两张表？

答案：`tz_prod_tag` 存分组本身，`tz_prod_tag_reference` 存商品和分组的关联。一个商品可以属于多个分组。
