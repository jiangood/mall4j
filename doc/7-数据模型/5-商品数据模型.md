# 商品数据模型

商品主表：

```text
tz_prod
```

SKU 表：

```text
tz_sku
```

## 商品表核心字段

| 字段 | 说明 |
| --- | --- |
| `prod_id` | 商品 ID。 |
| `prod_name` | 商品名称。 |
| `shop_id` | 店铺 ID。 |
| `ori_price` | 原价。 |
| `price` | 现价。 |
| `brief` | 简要描述。 |
| `content` | 商品详情，通常为富文本。 |
| `pic` | 商品主图。 |
| `imgs` | 商品图片，逗号分隔。 |
| `status` | 商品状态，`1` 正常，`0` 下架，`-1` 删除。 |
| `category_id` | 商品分类 ID。 |
| `sold_num` | 销量。 |
| `total_stocks` | 总库存。 |
| `delivery_mode` | 配送方式 JSON。 |
| `delivery_template_id` | 运费模板 ID。 |
| `version` | 乐观锁版本。 |

## SKU 表核心字段

| 字段 | 说明 |
| --- | --- |
| `sku_id` | SKU ID。 |
| `prod_id` | 所属商品 ID。 |
| `properties` | 销售属性组合，格式类似 `p1:v1;p2:v2`。 |
| `ori_price` | 原价。 |
| `price` | 售价。 |
| `stocks` | 库存相关字段。 |
| `actual_stocks` | 实际库存。 |
| `party_code` | 商家编码。 |
| `model_id` | 商品条形码。 |
| `pic` | SKU 图片。 |
| `sku_name` | SKU 名称。 |
| `prod_name` | 商品名称冗余。 |
| `version` | 乐观锁版本。 |
| `weight` | 重量。 |
| `volume` | 体积。 |
| `status` | `1` 启用，`0` 禁用。 |
| `is_delete` | `0` 正常，`1` 已删除。 |

## 关联关系

```text
tz_prod.prod_id = tz_sku.prod_id
```

一个商品可以有多个 SKU。下单和购物车最终都会落到具体 SKU。

## 规格相关表

```text
tz_prod_prop
tz_prod_prop_value
```

商品分组相关表：

```text
tz_prod_tag
tz_prod_tag_reference
```

## 商品分组

商品分组用于把商品挂到首页或店铺页的分组标签中。当前代码和 SQL 中对应实体：

```text
yami-shop-bean/src/main/java/com/yami/shop/bean/model/ProdTag.java
yami-shop-bean/src/main/java/com/yami/shop/bean/model/ProdTagReference.java
```

`tz_prod_tag` 核心字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 分组标签 ID。 |
| `shop_id` | 所属店铺 ID。 |
| `title` | 分组标题。 |
| `status` | `1` 正常，`0` 删除。 |
| `is_default` | `1` 系统默认，`0` 商家自定义。 |
| `prod_count` | 分组内商品数量统计。 |
| `seq` | 排序。 |
| `style` | 列表样式。 |

`tz_prod_tag_reference` 是商品和分组的关联表：

| 字段 | 说明 |
| --- | --- |
| `reference_id` | 关联 ID。 |
| `shop_id` | 所属店铺 ID。 |
| `tag_id` | 分组标签 ID。 |
| `prod_id` | 商品 ID。 |
| `status` | `1` 正常，`0` 删除。 |
| `create_time` | 创建时间。 |

一个商品可以关联多个商品分组。

## 完成后应该知道

- `tz_prod` 是商品主信息，`tz_sku` 是具体购买规格。
- 一个商品可以有多个 SKU。
- 购物车和下单最终都会落到具体 SKU。
- 商品分组由 `tz_prod_tag` 和 `tz_prod_tag_reference` 两张表完成。

## 检查题-含答案

问题：商品只有一种规格时，还需要 SKU 吗？

答案：需要。当前项目下单和购物车最终都会落到 SKU，即使只有一种规格，也应该有对应 SKU 记录。

问题：商品分组为什么不直接写在 `tz_prod` 上？

答案：因为一个商品可以属于多个分组，需要用 `tz_prod_tag_reference` 做多对多关联。
